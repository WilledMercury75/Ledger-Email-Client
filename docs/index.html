Capabilities

Privacy without compromise

Every message is sealed before it leaves your machine. Every route is a fallback. Nothing is left to chance.

✦

Sealed Cryptography

Ed25519 identity, X25519 key exchange, ChaCha20-Poly1305 authenticated encryption. The same primitives trusted by Signal and WireGuard.

⬡

P2P Delivery

Direct encrypted dispatch via libp2p. No central authority, no post office. Kademlia DHT stores messages until your recipient comes online.

◈

Gmail Bridge

When the direct route fails, your encrypted payload routes through Gmail — sealed so tightly that no intermediary can read it.

↯

Automatic Routing

P2P → DHT → Gmail. Your words find their audience by any route necessary. Choose auto, P2P-only, or Gmail-only modes.

⊞

Plugin Engine

Java-based message processing pipeline. Built-in spam filter, auto-tagger, and a simple API to write your own extensions.

⌘

Open API

Full REST API on port 8420. Build clients, bots, or integrations in any language. Every feature is an endpoint.

Architecture

Four languages. One republic.

Each component written in the language that serves it best — forged together through a unified API.

Interactive REPL, API client, and smoke test suite. Zero dependencies.

The Dispatches

JSON REST API on 127.0.0.1:8420

GET /api/identity Your sealed identity
GET /api/messages List by folder
POST /api/messages Send encrypted dispatch
DELETE /api/messages/{id} Burn after reading
GET /api/peers Connected couriers
POST /api/peers Connect via multiaddr
GET /api/settings Delivery mode & config
PUT /api/settings Update settings
POST /api/gmail/config Configure fallback post
POST /api/gmail/fetch Fetch from the post office
